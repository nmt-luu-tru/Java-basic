# Giới Thiệu về Lập Trình Hướng Đối Tượng (OOP) trong Java

Trong video này, chúng ta sẽ bắt đầu tìm hiểu về Lập Trình Hướng Đối Tượng (Object-Oriented Programming - OOP). OOP là một phương pháp để mô hình hóa các đối tượng trong thế giới thực thành các đối tượng phần mềm chứa cả dữ liệu và mã lệnh. Hãy cùng khám phá các khái niệm cơ bản của OOP, bao gồm lớp, đối tượng, các thành phần của lớp, và cách thực hiện chúng trong Java.

## 1. Lập Trình Hướng Đối Tượng là gì?

Lập Trình Hướng Đối Tượng (OOP) là một phương pháp lập trình dựa trên việc tổ chức phần mềm thành các đối tượng, mỗi đối tượng chứa cả dữ liệu (trạng thái) và các hành vi (phương thức) của nó. OOP giúp chúng ta mô hình hóa các đối tượng trong thế giới thực một cách linh hoạt và dễ hiểu.

### 1.1. Thành Phần Chính của OOP

- **Đối Tượng (Object):** Là một thực thể trong thế giới thực, ví dụ như máy tính, bàn phím, hoặc một chiếc xe hơi. Mỗi đối tượng có hai thành phần chính:
  - **Trạng Thái (State):** Các thuộc tính mô tả đặc điểm của đối tượng, chẳng hạn như màu sắc, kích thước, số lượng cửa của một chiếc xe hơi.
  - **Hành Vi (Behavior):** Các phương thức mà đối tượng có thể thực hiện, chẳng hạn như khởi động, dừng lại, hoặc thay đổi màu sắc.

- **Lớp (Class):** Là bản thiết kế hoặc khuôn mẫu để tạo ra các đối tượng. Lớp định nghĩa các thuộc tính và phương thức mà các đối tượng của lớp đó sẽ có.

## 2. Lớp và Đối Tượng trong Java

### 2.1. Định Nghĩa Lớp

Lớp trong Java được định nghĩa bằng từ khóa `class`. Dưới đây là một ví dụ về cách tạo một lớp đơn giản gọi là `Car`:

```java
public class Car {
    private String make;
    private String model;
    private String color;
    private int doors;
    private boolean convertible;

    public void describeCar() {
        System.out.println("Number of doors: " + doors);
        System.out.println("Color: " + color);
        System.out.println("Make: " + make);
        System.out.println("Model: " + model);
        System.out.println("Convertible: " + (convertible ? "Yes" : "No"));
    }
}
```

### 2.2. Thành Phần của Lớp

- **Trường (Fields):** Các biến trong lớp lưu trữ trạng thái của đối tượng. Trong ví dụ trên, `make`, `model`, `color`, `doors`, và `convertible` là các trường.
- **Phương Thức (Methods):** Các hàm trong lớp định nghĩa hành vi của đối tượng. Trong ví dụ, `describeCar` là một phương thức mô tả thông tin về xe hơi.

### 2.3. Tạo Đối Tượng từ Lớp

Để tạo một đối tượng từ lớp `Car`, chúng ta sẽ sử dụng từ khóa `new`:

```java
public class Main {
    public static void main(String[] args) {
        Car myCar = new Car();
        myCar.describeCar();
    }
}
```

## 3. Các Trình Điều Khiển Truy Cập (Access Modifiers)

### 3.1. Các Loại Trình Điều Khiển Truy Cập

- **Public:** Cho phép truy cập từ bất kỳ đâu.
- **Private:** Chỉ cho phép truy cập từ bên trong lớp.
- **Protected:** Cho phép truy cập từ các lớp con và trong cùng một gói.
- **Không Có Modifier (Package-Private):** Chỉ cho phép truy cập từ các lớp trong cùng một gói.

### 3.2. Bảo Vệ Trường và Phương Thức

Thường thì các trường trong lớp được khai báo là `private` để bảo vệ dữ liệu và chỉ cho phép truy cập thông qua các phương thức `getter` và `setter`.

```java
public class Car {
    private String make;
    private String model;
    private String color;
    private int doors;
    private boolean convertible;

    // Getter và Setter cho make
    public String getMake() {
        return make;
    }

    public void setMake(String make) {
        this.make = make;
    }

    // Các getter và setter khác tương tự
}
```

## 4. Đóng Gói (Encapsulation)

Đóng gói là một nguyên tắc quan trọng của OOP, bao gồm:

- **Bundling (Gói Gọn):** Kết hợp dữ liệu và phương thức trong một đối tượng.
- **Information Hiding (Ẩn Thông Tin):** Ẩn các chi tiết bên trong của đối tượng khỏi bên ngoài để bảo vệ tính toàn vẹn của dữ liệu.

Việc đóng gói giúp mã nguồn trở nên dễ bảo trì và bảo mật hơn, đồng thời giảm sự phụ thuộc giữa các phần của chương trình.

## 5. Tạo và Sử Dụng Lớp `Car` Thực Tế

### 5.1. Khởi Tạo Các Trường trong Lớp `Car`

Trong lớp `Car`, chúng ta đã định nghĩa các trường để lưu trữ thông tin về xe hơi:

```java
public class Car {
    private String make;
    private String model;
    private String color;
    private int doors;
    private boolean convertible;

    // Phương thức mô tả xe hơi
    public void describeCar() {
        System.out.println("Number of doors: " + doors);
        System.out.println("Color: " + color);
        System.out.println("Make: " + make);
        System.out.println("Model: " + model);
        System.out.println("Convertible: " + (convertible ? "Yes" : "No"));
    }
}
```

### 5.2. Tạo Phương Thức Mô Tả Xe Hơi

Phương thức `describeCar` sẽ in ra thông tin về xe hơi:

```java
public void describeCar() {
    System.out.println("Number of doors: " + doors);
    System.out.println("Color: " + color);
    System.out.println("Make: " + make);
    System.out.println("Model: " + model);
    System.out.println("Convertible: " + (convertible ? "Yes" : "No"));
}
```

### 5.3. Sử Dụng Lớp `Car` trong Lớp `Main`

Trong lớp `Main`, chúng ta sẽ tạo một đối tượng `Car` và sử dụng phương thức `describeCar` để hiển thị thông tin về xe:

```java
public class Main {
    public static void main(String[] args) {
        Car myCar = new Car();
        myCar.setMake("Toyota");
        myCar.setModel("Camry");
        myCar.setColor("Red");
        myCar.setDoors(4);
        myCar.setConvertible(false);
        
        myCar.describeCar();
    }
}
```

### 5.4. Kết Quả Khi Chạy Chương Trình

Khi chạy chương trình, kết quả sẽ là:

```
Number of doors: 4
Color: Red
Make: Toyota
Model: Camry
Convertible: No
```

## 6. Tổ Chức Các Lớp vào Gói (Packages)

Các lớp trong Java thường được tổ chức vào các gói để quản lý mã nguồn một cách hiệu quả. Để khai báo một lớp thuộc về một gói, chúng ta sử dụng từ khóa `package`:

```java
package com.example.vehicles;

public class Car {
    // Nội dung lớp
}
```

Nếu không khai báo gói, lớp sẽ thuộc về gói mặc định (default package).

## 7. Tầm Quan Trọng của Lớp trong Lập Trình

### 7.1. Lớp như Một Kiểu Dữ Liệu Người Dùng Định Nghĩa

Lớp có thể được xem như một kiểu dữ liệu mạnh mẽ do người dùng định nghĩa, cho phép chúng ta tạo ra các đối tượng với các thuộc tính và hành vi tùy chỉnh.

### 7.2. Lợi Ích của OOP

- **Tái Sử Dụng Mã Nguồn:** Các lớp có thể được tái sử dụng trong nhiều phần của chương trình.
- **Bảo Trì Dễ Dàng:** Các thay đổi trong lớp không ảnh hưởng đến các phần khác của chương trình.
- **Mô Hình Hóa Thực Tế:** OOP giúp mô hình hóa các đối tượng trong thế giới thực một cách trực quan và dễ hiểu.
